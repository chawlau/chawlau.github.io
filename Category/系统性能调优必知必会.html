协程

高并发实现条件

由内核实现的请求切换工作，交由用户态的代码来完成就可以了

为什么必须非阻塞

协程必须重新封装所有的阻塞系统调用，否则，一旦协程触发了线程切换，会导致这个线程进入休眠状态，进而其上的所有协程都得不到执行

协程切换

  • 就是保存SP&IP的状态

协程的代价

线程的栈有 8MB，而协程栈的大小通常只有几十 KB。而且，C 库内存池也不会为协程预分配内存，它感知不到协程的存在。这样，更低的内存占用空间为高并发提供了保证，毕竟十万并发请求，就意味着 10 万个协程

提升tcp三次握手的性能

客户端的优化

  • tcp_syn_retries 根据网络稳定性和目标服务器繁忙程度修改重试次数，适当调低重试次数，尽快把错误暴露给应用程序
  • 重试6次，总共2分钟

服务器端的优化

  • 调大半连接队列，tcp_max_syn_backlog
  • 如果半连接满了，开启 syncookies 功能就可以在不使用 SYN 队列的情况下成功建立连接，tcp_syncookies = 1 表示队列满的时候使用
  • 调整sync_ack重试次数 tcp_synack_retries
  • 如果accept队列溢出，tcp_abort_on_overflow = 1 告诉客户端已经失败
  • accept队列大小 net.core.somaxconn

绕过三次握手

  • 第一阶段为首次建立连接，这时走正常的三次握手，但在客户端的 SYN 报文会明确地告诉服务器它想使用 TFO 功能，这样服务器会把客户端 IP 地址用只有自己知道的密钥加密（比如 AES 加密算法），作为 Cookie 携带在返回的 SYN+ACK 报文中，客户端收到后会将 Cookie 缓存在本地。
  • 如果客户端再次向服务器建立连接，就可以在第一个 SYN 报文中携带请求数据，同时还要附带缓存的 Cookie，需要使用send_to&sendmsg
  • net.ipv4.tcp_fastopen = 3

如何提升tcp四次挥手的性能

被动方如何关闭

  • CLOSE_WAIT 状态时，进程的 read 函数会返回 0
  • 开发人员就会有针对性地调用 close 函数，然后状态变成LAST_ACK

主动方的优化

  • close 和 shutdown都会发FIN报文

孤儿连接

  • 当进程调用 close 函数关闭连接后，无论该连接是在 FIN_WAIT1 状态，还是确实关闭了，这个连接都与该进程无关了，它变成了孤儿连接
  • tcp_max_orphans 最大孤儿连接
  • tcp_fin_timeout = 60s
  • tcp_orphan_retries 控制

TimeWait

  • 连接的端口恢复了自由身，可以复用于新连接了
  • 被动方的 FIN 报文可能再次到达，这既可能是网络中的路由器重复发送
  • 正常通讯的新连接就可能被重复发送的 FIN 报文误关闭

为什么是2MSL

  • 这其实是相当于至少允许报文丢失一次。比如，若 ACK 在一个 MSL 内丢失，这样被动方重发的 FIN 会在第 2 个 MSL 内到达，TIME_WAIT 状态的连接可以应对

参数

  • tcp_max_tw_buckets，TIME_WAIT 的连接数量超过该参数时，新关闭的连接就不再经历 TIME_WAIT 而直接关闭
  • tcp_tw_reuse=1 服务器主动向上游服务器发起连接可以复用 需要tcp_timestamps=1 来设置

被动方的优化

  • 大量close_wait
  • 要么是程序出现了 Bug，read 函数返回 0 时忘记调用 close 函数关闭连接
  • close 程序负载太高，close 函数所在的回调函数被延迟执行了

同时发起FIN

  • 两边都处于CLOSING，类似于LAST_ACK
  • 接收ACK就关闭

如何调整TCP拥塞控制技能

  • 通过定时器明确探测到了丢包；
  • 拥塞窗口的增长到达了慢启动阈值 ssthresh（全称为 slow start threshold），也就是之前发现网络拥塞时的窗口大小；
  • 接收到重复的 ACK 报文，可能存在丢包。

测量带宽时延

  • 测量带宽、时延来确定拥塞的方法，在丢包率较高的网络中应用效果尤其好

CPU缓存

CPU缓存大小

  • 一级缓存是32KB
  • 二级缓存是256KB
  • 三级缓存是20MB
  • 每个核心都有自己的一、二级缓存，但三级缓存却是一颗 CPU 上所有核心共享的

    CPU 访问一次内存通常需要 100 个时钟周期以上，而访问一级缓存只需要 4~5 个时钟周期，二级缓存大约 12 个时钟周期，三级缓存大约 30 个时钟周期

Cache line

  • 内核中coherency_line_size定义
  • nginx的哈希表桶的大小等于Cache line
  • 按照 cpu cache line（比如 64 字节）来访问内存时，不会出现多核 CPU 下的伪共享问题，可以尽量减少访问内存的次数

显式预测分支概率

  • likely&unlikely

思考题

  • 多线程并行访问不同的变量，这些变量在内存布局是相邻的，此时 CPU 缓存就会失效，为什么？
  • 一片连续的内存被加载到不同cpu核心中（就是同一个cache line在不同的cpu核心），其中一个cpu核心中修改cache line,其它核心都失效，加锁也是加在cache line上，其它核心线程也被锁住，降低了性能。解决办法是填充无用字节数，使分开

GEO

二分区间，区间编码

  • 经度范围[-180,180]会被分成两个子区间
  • 如果是落在左分区，我们就用 0 表示；如果落在右分区，就用 1 表示
  • 维度范围[-90, 90]会被分成两个子区间
  • 如果是落在左分区，我们就用 0 表示；如果落在右分区，就用 1 表示

组合规则

  • 一组经纬度值都编完码后，我们再把它们的各自编码值组合在一起，组合的规则是：最终编码值的偶数位上依次是经度的编码值，奇数位上依次是纬度的编码值
  • 有的编码值虽然在大小上接近，但实际对应的方格却距离比较远

操作指令

  • GEOADD
  • GEORADIUS
