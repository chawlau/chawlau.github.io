  • 基于tcp协议开发提供服务的应用程序，提供的一层的访问控制工具
  • 基于库调用实现功能

判断服务是否能够由tcp_wrapper进行访问控制

  • 动态编译: ldd命令查看是否有libwrap.so
  • 静态编译 strings命令查看应用程序，并且其结果中如果出现host.allow, host.deny

配置文件实现访问控制

  • /etc/hosts/allow
  • /etc/hosts/deny
  • 配置文件语法：
  • daemon_list: client_list [:options]
daemon_list:
  • 应用程序的文件名称，而非服务名
  • 应用程序文件名称列表，逗号分隔
  • 例如 sshd, vsftpd
  • ALL表示所有
client_list
  • IP地址
  • 主机名
  • 网络地址
  • 简短格式的网络格式
  • ALL表示所有主机
  • KNOW
  • UNKNOW
  • PARANOID

[:options]

  • spawn: 启动额外应用程序

Iptable之内网访问互联网

NAT，proxy

  • nat 隐藏本地网络的IP地址 Network Address Translation 网络层+传输层
  • proxy 代理 应用层

nat技术分类

  • SNAT 只修改请求报文的源地址
  • DNAT 只修改请求报文的目标地址
  • IPVS 基于10种方式来实现负载均衡LVS
  iptables -t nat -A PREROUTING -d gateway -p tcp --dport 80 -j DNAT --to-destination dest_ip:port
  iptables -t nat -A PREROUTING -d gateway -p tcp --dport 22 -j DNAT --to-destination dest_ip:port 端口映射
  iptables -t nat -A POSTROUTING -s 192.168.20.0/24 ! -d 192.168.20.0/24 -j MASQUERADE //地址伪装

NAT表

  • PREROUTING: DNAT
  • OUTPUT
  • POSTROUTING: SNAT
  • REDIRECT 端口重定向
  • 美国往事//TODO

nat的实现方式

  • 源地址转化 iptables -t nat -A POSTROUTING -s LOCALNET ! -d LOCALNET -j SNAT --to-source ExtIP
  • iptables -t nat -A POSTROUTING -s LOCALNET ! -d LOCALNET -j MASQUERADE 地址不固定
  • iptables -t nat -L -n
  • 目标地址转换 iptables -t nat -A PREROUTING -d ExtIP -p tcp --dport PORT -j DNAT --to-destination InterServerIP

Mysql监控

  • MySQL 数据库我们都要监控什么
  • 怎么对这些要监控的资源进行监控

监控可用性

  • 对数据库服务可用性进行监控
如何确认数据库是否可以通过网络连接
mysqladmin-umonitor_user-p h ping
telnet ip port
应用程序通过网络建立数据库连接
  • 可以连接到数据库不代表可用
如何确认数据库是否可读写
  • 检査数据库的 read_only 参数是否为 off
  • 建立监控表并对表中数据进行更行执行简单的查询
  • select@@ version
如何监控数据库的连接数
  • 时刻关注数据库连接数量的变化
show variables like 'max connections 
show global status like 'Threads_connected'
threads connected/max connections 0.8

对数据库性能进行监控

  • 记录性能监控过程中所采集到的数据库的状态
如何计算QPS和TPS
* QPS=(Queries2-Queries1)(差值)/
(Uptime_since_flush_status2- Uptime_since_flush_status1）(时间间隔)
* TPS=((Com_insert+Com_update2+Com_delete
 (Com_insert1+Com_update1+Com_deletel))/
 (Uptimesince_flushStatus2-uptime_since_flush_statusi)
如何监控数据库的并发请求数量
show global status like 'Threads_running'
  • 数据库系统的性能会随着并发处理请求数量的增加而下降
如何监控Innodb阻塞

  • information_schema.innodb_lock_waits
  • information_schema.innodb_trx
select connection_id() 查看连接ID

主从复制链路状态的监控

  • show slave status
