尾部递归调用

优化

  • C 语言默认情况下是不考虑并发的，因为并发是自己处理的，那么 GCC 编译器尽可能在指令上做到性能优先，这是 GCC 目标。Go 语言是指令尽可能优化，前提是 runtime 机制优先，runtime 机制优先于算法优化，所以避免把多次函数调用优化成时间非常长的循环。
  • GCC 考虑的是 CPU 密集型。Go 考虑的是多任务的公平性

defer工作机制

  • 第一个是函数，第二个是所有参数，它会立即求值，把参数和函数一起打包放在一个地方，等着后面去执行
  • derfer 语句实际上形成两个函数调用，第一个是 derferproc 注册，把函数和参数打包放在一个地方不执行，然后在 RET 之前通过 deferreturn 函数来调用注册的函数

延迟函数一定被执行

func main() {
    defer println("exit")
    //os.Exit(0)
    panic("abc")
}
  • 引发 panic，延迟函数也会被执行。
  • 用 runtime.Goexit 终止当前 goroutine，延迟函数会被执行。
  • 但os.Exit 会立即终止进程，不执行任何延迟函数。

闭包

  • 在堆上创建数据结构，这个数据结构有两个字段组成的，接下来把匿名函数的地址填到第一个字段里面，把环境变量的值拷贝到第二个字段里面。这个结构体包含闭包执行需要的数据，所以匿名函数加上所需要的环境变量叫做闭包。
  • main 函数怎么执行这个匿名函数呢，首先通过 SP 拿到返回值，赋值给 DX，DX 赋值给局部变量 f，所以 f 严格意义上来说不是一个简单的函数，是闭包的第一个字段。DX 读出一个值放到 AX 内，接下来直接 call AX 相当于调用匿名函数，这个匿名函数通过 DX+8 取到第二个字段，就构成了一套的完整的闭包调用
  • 包装对象只有匿名函数地址，没有环境变量指针，因为匿名函数地址是固定的，所以这个包装对象不需要每次生成，编译器把它写到 rodata 里面。如果有闭包的话，环境变量的地址在堆上分配的，那么这个地址肯定是在运行期动态生成的，那么包装对象就没有办法在编译期确定，所以这个包装对象就不能生成到 rodata 里面。

迭代器模式

动机

  • 在软件构建过程中，集合对象内部结构常常变化各异。但对于这些集合对象，我们希望在不暴露其内部结构的同时，可以让外部客户代码透明地访词其中包含的元素；同时这种“透明遍历”也为同一种算法在多种集合对象上进行操作”提供了可能。

  • 使用面向对象技术将这种遍历机制抽象为“迭代器对象”为“应对变化中的集合对象”提供了一种优雅的方式。

要点总结

  • 迭代抽象：访问一个聚合对象的内容而无需暴露它的内部表示。迭代多态：为遍历不同的集合结构提供一个统一的接口，从而支
  • 持同样的算法在不同的集合结构上进行操作。
  • 迭代器的健壮性考虑：遍历的同时更改迭代器所在的集合结构，会导致问题。

可重入的锁

  • 同一个线程里面，可以连续调用多次acquire,一定要注意acquire的次数
  • RLock可重入锁
from threading import Lock, RLock, Condition

total = 0
lock = RLock()

def add():
    global lock
    global total
    for i in range(10000):
        lock.acquire()
        lock.acquire()
        total += 1
        lock.release()
        lock.release()

def desc():
    global total
    global lock
    for i in range(10000):
        # lock.acquire()
        lock.acquire()
        total -= 1
        lock.release()
        # lock.release()

import threading

thread1 = threading.Thread(target=add)
thread2 = threading.Thread(target=desc)

thread1.start()
thread2.start()
thread2.join()
thread2.join()
print("total {}".format(total))

class XiaoAi(threading.Thread):
    def __init__(self, lock):
        super().__init__(name="xiaoai")
        self.lock = lock

    def run(self):
        self.lock.acquire()
        print("{}. at ".format(self.name))
        self.lock.release()

        self.lock.acquire()
        print("{}. ok ".format(self.name))
        self.lock.release()

    def __init__(self, lock):
        super().__init__(name="tiaomao")
        self.lock = lock

        self.lock.acquire()
        print("{} : xiaoai roomate".format(self.name))
        self.lock.release()

        self.lock.acquire()
        print("{} : let's talk".format(self.name))
        self.lock.release()

    thread2.join()

import threading

class XiaoAi(threading.Thread):
    def __init__(self, cond):
        super().__init__(name="xiaoai")
        self.cond = cond

    def run(self):
        with self.cond:
            self.cond.wait()
            print('{}: 在 '.format(self.name))
            self.cond.notify()

            self.cond.wait()
            print('{}: 好阿 '.format(self.name))
            self.cond.notify()

            self.cond.wait()

    return times

executor = ThreadPoolExecutor(max_workers=2)

urls = [3, 2, 4]

all_task = [executor.submit(get_html, (url)) for url in urls]

GIL

概念

  • 全局解释器锁
  • CPython
  • GIL使得同一时刻只有一个线程在一个CPU上执行字节码
  • 无法将多个线程映射到多个CPU上执行

释放的时机

  • 根据执行字节码的行数或者执行的时间片来进行释放
  • 遇到IO操作的时候释放

线程调度

  • 对于IO操作来说，多线程和多进程性能差别不大

集成Thread来实现多线程

import time
import threading

class GetDetailHtml(threading.Thread):
    def __init__(self, name):
        super().__init__(name=name)

    def run(self):
        print("get detail html started")
        time.sleep(2)
        print("get detail html end")

class GetDetailUrl(threading.Thread):
    def __init__(self, name):
        super().__init__(name=name)

    def run(self):
        print("get detail url started")
        time.sleep(4)
        print("get detail url end")

if __name__ == "__main__":
    thread1 = GetDetailHtml("get_detail_html")
    thread2 = GetDetailUrl("get_detail_url")
    start_time = time.time()
    thread1.start()
    thread2.start()
    thread1.join()
    thread2.join()

    print("last time : {}".format(time.time() - start_time))
