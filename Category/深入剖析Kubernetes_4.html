深入理解容器镜像

mount space

  • Mount Namepace修改的是容器进程对文件系统挂载点的认知
  • tmpfs内存盘

挂载原理

  mount(“”, “/“, NULL, MS_PRIVATE, “”)

chroot

  • 改变进程的根目录到指定的位置
  • 挂载在容器根目录上，用来为容器进程提供隔离后执行环境的文件系统就是容器镜像，也叫rootfs

Docker核心流程

  • 启动linux namespace配置
  • 设置指定的cgroups参数
  • 切换进程的根目录

pivot_root和chroot的区别

容器一致性

  • rootfs有层级概念
  • 每一层对应一个文件夹
  • OverlayFS仅有两层，也就是说镜像中的每一层并不对应OverlayFS中的层，而是，镜像中的每一层对应/var/lib/docker/overlay中的一个文件夹，文件夹以该层的UUID命名。然后使用硬连接将下面层的文件引用到上层。这在一定程度上节省了磁盘空间。这样，OverlayFS中的lowerdir就对应镜像层的最上层，并且是只读的。在创建镜像时，Docker会新建一个文件夹作为OverlayFS的upperdir，它是可写的。

删除文件

  • 通过可读写层创建一个whiteout文件，把只读层的文件遮挡起来

init层

  • Init 层是 Docker 项目单独生成的一个内部层，专门用来存放 /etc/hosts、/etc/resolv.conf 等信息。

容器修改文件的流程

  • 修改一个文件的时候，首先会从上到下查找有没有这个文件，找到，就复制到容器层中，修改，修改的结果就会作用到下层的文件，这种方式也被称为copy-on-write。

katacontainers

  • 新的容器技术//TODO

exec原理

  • docker inspect --format '{{.State.Pid}}' container-id
  • 通过加入一个进程的NameSpace中达到进入容器的目的
  • 通过调用sentns的系统调用实现

容器里进程新建的文件，怎么才能让宿主机获取到？

  • rootfs之后,chroot之前，宿主机的目录被联合挂载到/var/lib/docker/aufs/mnt/目录中的可读写层中，这个挂载工作就完成了
  • 容器进程是指docker创建的容器初始化进程，最后通过execv()系统调用，让应用进程取代自己，成为容器pid=1的进程
  • 绑定挂载允许将一个目录或者文件挂载到指定的目录上，该挂载点上进行的任何操作，只是发生在被挂载的目录或者文件上，而原挂载点的内容则会被隐藏起来且不受影响。
  • 绑定挂载本质就是inode替换的过程
  • 新建目录不会被namespace隔离，会产生新的目录，但是对该目录的改变只会产生在宿主机，不会在容器中产生。

Docker copyData功能

cgroup的NameSpace作用

  • 容器中一旦挂载cgroup filesystem，可以修改整全局的cgroup配置
  • cgroup namespace更加安全。
  • namespace中的进程都有自己cgroupns root和cgroup filesystem视图
  • 会使容器迁移更加容易；在迁移时，/proc/self/cgroup需要复制到目标机器，这要求容器的cgroup路径是唯一的，否则可能会与目标机器冲突

  • mount -t overlay overlay -olowerdir=./lower,upperdir=./upper,workdir=./work ./merged

  • 基于tcp协议开发提供服务的应用程序，提供的一层的访问控制工具

  • /etc/hosts/deny

  • daemon_list: client_list [:options]

  • 应用程序的文件名称，而非服务名

  • 例如 sshd, vsftpd
  • ALL表示所有

  • IP地址
  • 主机名
  • 网络地址

  • ALL表示所有主机

Iptable之内网访问互联网

NAT，proxy

  • nat 隐藏本地网络的IP地址 Network Address Translation 网络层+传输层
  • proxy 代理 应用层

nat技术分类

  • SNAT 只修改请求报文的源地址
  • DNAT 只修改请求报文的目标地址
  • IPVS 基于10种方式来实现负载均衡LVS
  iptables -t nat -A PREROUTING -d gateway -p tcp --dport 80 -j DNAT --to-destination dest_ip:port
  iptables -t nat -A PREROUTING -d gateway -p tcp --dport 22 -j DNAT --to-destination dest_ip:port 端口映射
  iptables -t nat -A POSTROUTING -s 192.168.20.0/24 ! -d 192.168.20.0/24 -j MASQUERADE //地址伪装

NAT表

  • PREROUTING: DNAT
  • OUTPUT
  • POSTROUTING: SNAT
  • REDIRECT 端口重定向
  • 美国往事//TODO

nat的实现方式

  • 源地址转化 iptables -t nat -A POSTROUTING -s LOCALNET ! -d LOCALNET -j SNAT --to-source ExtIP
  • iptables -t nat -A POSTROUTING -s LOCALNET ! -d LOCALNET -j MASQUERADE 地址不固定
  • iptables -t nat -L -n
  • 目标地址转换 iptables -t nat -A PREROUTING -d ExtIP -p tcp --dport PORT -j DNAT --to-destination InterServerIP
